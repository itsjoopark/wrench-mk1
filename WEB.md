# Wrench — Next.js frontend

A mobile-friendly web UI for the Wrench bike copilot, powered by Perceptron's
Mk1 VLM. Replaces the Gradio Hugging Face Space. Two modes:

- **Anatomy** — a fixed demo bike; tap a part to see a close-up crop and ask
  Mk1 what it does.
- **Diagnose** — upload/photograph your own bike, describe a symptom, get a
  schema-validated repair plan.

## Architecture

- **Next.js (App Router)** at the repo root — the UI (`app/`, `components/`, `lib/`).
- **Python serverless functions** in `api/` — thin wrappers that reuse the
  existing `wrench/` package (detection, VQA, structured diagnosis). Deployed on
  Vercel alongside the Next.js app in one project.

The browser sends images as base64 to `/api/anatomy`, `/api/ask`, `/api/diagnose`.
The `PERCEPTRON_API_KEY` lives only on the server — never shipped to the client.

```
app/            Next.js pages (/, /tool) — landing + tool
components/      UI components (modes, close-up, repair plan, ...)
lib/            TS types + constants (mirror wrench/schema.py + config.py)
api/*.py        Vercel Python functions (reuse wrench/)
wrench/         existing model logic (unchanged)
scripts/        precompute_anatomy.py, make_placeholder.py
public/         bike.png (demo bike) + anatomy-boxes.json (precomputed)
```

## Local development

```bash
npm install
python3 -m venv .venv && source .venv/bin/activate
pip install -r api/requirements.txt

# Add your key to .env (copy from .env.example) — must be saved to disk
cp .env.example .env   # then edit PERCEPTRON_API_KEY=...

# Full stack (UI + Python API via local proxy):
npm run dev:full       # http://localhost:3000

# UI only (demo bike works; Ask Mk1 / Diagnose need dev:full):
npm run dev
```

`npm run dev:full` runs two processes:
1. `scripts/local_api.py` on port 5328 (Python, reads `.env`)
2. Next.js with `USE_LOCAL_API=1` rewrites `/api/*` to the local Python server

Alternative: `npm run dev:vercel` (`vercel dev`) for Vercel's Python routing on deploy parity.

Set `PERCEPTRON_API_KEY` in `.env` first. **Save the file** (Cmd+S) — unsaved editor buffers are not read by the Python server.

## The demo bike asset

`public/bike.png` currently holds a **placeholder** (generated by
`scripts/make_placeholder.py`). Replace it with the real S-Works bike exported
from Figma (transparent PNG recommended), then regenerate the cached detections:

```bash
pip install -r api/requirements.txt
PERCEPTRON_API_KEY=sk-... python scripts/precompute_anatomy.py
```

That overwrites `public/anatomy-boxes.json` with real, pixel-accurate boxes.

## Deploy (Vercel)

1. Import the repo into Vercel (framework auto-detected as Next.js).
2. Add the `PERCEPTRON_API_KEY` environment variable.
3. Deploy. `vercel.json` bundles `wrench/**` into each Python function.
